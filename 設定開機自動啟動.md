# 設定開機自動啟動開發伺服器

## 方法一：使用啟動資料夾（推薦）

### 步驟：

1. **建立啟動腳本捷徑**
   - 按 `Win + R` 開啟執行對話框
   - 輸入：`shell:startup`
   - 按 Enter 開啟啟動資料夾

2. **建立捷徑**
   - 在啟動資料夾中，右鍵 → 新增 → 捷徑
   - 瀏覽到：`C:\Web\drservWeb\start-dev-server.bat`
   - 或直接將 `start-dev-server.bat` 複製到啟動資料夾

3. **完成**
   - 重新開機後，開發伺服器會自動啟動
   - 會開啟一個 PowerShell 視窗顯示伺服器狀態

## 方法二：使用工作排程器（更進階）

### 步驟：

1. **開啟工作排程器**
   - 按 `Win + R`，輸入 `taskschd.msc`
   - 或搜尋「工作排程器」

2. **建立基本工作**
   - 右側點擊「建立基本工作」
   - 名稱：`捨得資訊開發伺服器`
   - 描述：`開機自動啟動網站開發伺服器`

3. **設定觸發程序**
   - 選擇「當我的電腦啟動時」

4. **設定動作**
   - 選擇「啟動程式」
   - 程式或指令碼：`C:\Web\drservWeb\start-dev-server.bat`
   - 開始於：`C:\Web\drservWeb`

5. **完成設定**
   - 勾選「開啟 [內容] 對話方塊...」
   - 完成後，在「一般」標籤中：
     - ✅ 勾選「以最高權限執行」（如果需要）
     - ✅ 勾選「不論使用者登入與否都要執行」（可選）

6. **測試**
   - 右鍵點擊建立的工作 → 執行
   - 確認開發伺服器正常啟動

## 方法三：使用服務（進階，需要管理員權限）

如果需要以 Windows 服務方式運行，可以使用 `node-windows` 或 `pm2`：

```powershell
# 安裝 pm2
npm install -g pm2
npm install -g pm2-windows-startup

# 設定開機啟動
pm2 start npm --name "drserv-website" -- run dev
pm2 save
pm2 startup
```

## 注意事項

1. **防火牆設定**
   - 首次啟動時，Windows 可能會詢問是否允許 Node.js 通過防火牆
   - 請選擇「允許存取」

2. **自動開啟瀏覽器**
   - 開發伺服器啟動後會自動開啟瀏覽器
   - 如果不想自動開啟，可以修改 `vite.config.ts` 中的 `open: false`

3. **停止伺服器**
   - 在工作管理員中找到 Node.js 進程並結束
   - 或在 PowerShell 視窗中按 `Ctrl + C`

4. **查看日誌**
   - 伺服器輸出會顯示在 PowerShell 視窗中
   - 可以查看是否有錯誤訊息

## 快速測試

在設定自動啟動前，可以先手動測試腳本：

```powershell
cd C:\Web\drservWeb
.\start-dev-server.ps1
```

如果正常啟動，就可以設定為開機自動啟動了。

## 移除自動啟動

如果不想再自動啟動：

1. **啟動資料夾方式**：刪除啟動資料夾中的捷徑
2. **工作排程器方式**：在工作排程器中刪除對應的工作
