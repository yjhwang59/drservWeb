# 編碼設定說明

## 📝 概述

本專案已設定為使用 **UTF-8 編碼**，確保繁體中文和其他多語言字元能正確顯示。

## ✅ 已設定的檔案

### 1. `.editorconfig`
- 統一所有檔案的編碼為 UTF-8
- 設定行尾符號和縮排規則
- 適用於所有支援 EditorConfig 的編輯器（包括 Cursor、VSCode）

### 2. `.vscode/settings.json`
- 設定 Cursor/VSCode 的檔案編碼為 UTF-8
- 自動偵測檔案編碼
- 針對不同檔案類型（Markdown、TypeScript、JSON 等）設定 UTF-8

## 🔧 如何確認編碼設定

### 在 Cursor/VSCode 中檢查

1. **查看檔案編碼**
   - 開啟檔案後，查看右下角狀態列
   - 應該顯示 `UTF-8` 或 `UTF-8 with BOM`
   - 如果顯示其他編碼（如 `Big5`、`GB2312`），點擊後選擇「透過編碼儲存」→ 選擇 `UTF-8`

2. **設定預設編碼**
   - 檔案 → 喜好設定 → 設定
   - 搜尋 `files.encoding`
   - 確認設定為 `utf8`

### 在 PowerShell 中檢查檔案編碼

```powershell
# 檢查檔案編碼
Get-Content "部署指南-Cloudflare與防火牆設定.md" -Encoding UTF8 | Out-Null
Write-Host "檔案編碼檢查完成"

# 如果檔案編碼不正確，可以轉換
$content = Get-Content "檔案名稱.md" -Encoding Big5
$content | Set-Content "檔案名稱.md" -Encoding UTF8
```

## 📄 建立新檔案時的注意事項

### 建立 Markdown 檔案

1. **使用 Cursor/VSCode 建立**
   - 檔案會自動使用 UTF-8 編碼（根據 `.vscode/settings.json`）

2. **使用 PowerShell 建立**
   ```powershell
   # 正確方式：指定 UTF-8 編碼
   "內容" | Out-File -FilePath "檔案名稱.md" -Encoding UTF8
   
   # 或使用 Set-Content（預設 UTF-8）
   "內容" | Set-Content -Path "檔案名稱.md" -Encoding UTF8
   ```

3. **避免使用**
   ```powershell
   # ❌ 錯誤：可能使用系統預設編碼（Big5）
   "內容" | Out-File "檔案名稱.md"
   ```

### 建立程式碼檔案

- TypeScript/JavaScript：自動使用 UTF-8（根據 `.editorconfig`）
- JSON：自動使用 UTF-8
- HTML/CSS：自動使用 UTF-8

## 🔍 檢查檔案是否為 UTF-8

### 使用 PowerShell

```powershell
# 檢查檔案編碼
function Get-FileEncoding {
    param([string]$FilePath)
    $bytes = [System.IO.File]::ReadAllBytes($FilePath)
    if ($bytes.Length -ge 3 -and $bytes[0] -eq 0xEF -and $bytes[1] -eq 0xBB -and $bytes[2] -eq 0xBF) {
        return "UTF-8 with BOM"
    }
    try {
        $content = [System.IO.File]::ReadAllText($FilePath, [System.Text.Encoding]::UTF8)
        $bytes2 = [System.Text.Encoding]::UTF8.GetBytes($content)
        if ([System.Linq.Enumerable]::SequenceEqual($bytes, $bytes2)) {
            return "UTF-8"
        }
    } catch {}
    return "Unknown or not UTF-8"
}

# 使用範例
Get-FileEncoding "部署指南-Cloudflare與防火牆設定.md"
```

### 使用線上工具

- [Encoding Detector](https://www.webtoolkitonline.com/encoding-detector.html)
- 上傳檔案或貼上內容，自動偵測編碼

## 🛠️ 轉換現有檔案為 UTF-8

如果發現現有檔案不是 UTF-8 編碼，可以使用以下方法轉換：

### PowerShell 腳本

```powershell
# 轉換單一檔案
function Convert-ToUTF8 {
    param(
        [string]$FilePath,
        [string]$SourceEncoding = "Big5"
    )
    
    $content = Get-Content -Path $FilePath -Encoding $SourceEncoding -Raw
    [System.IO.File]::WriteAllText($FilePath, $content, [System.Text.Encoding]::UTF8)
    Write-Host "已轉換: $FilePath" -ForegroundColor Green
}

# 使用範例
Convert-ToUTF8 -FilePath "部署指南-Cloudflare與防火牆設定.md" -SourceEncoding "Big5"

# 批次轉換目錄中所有 .md 檔案
Get-ChildItem -Path "." -Filter "*.md" | ForEach-Object {
    try {
        $content = Get-Content -Path $_.FullName -Encoding UTF8 -Raw
        [System.IO.File]::WriteAllText($_.FullName, $content, [System.Text.Encoding]::UTF8)
        Write-Host "已轉換: $($_.Name)" -ForegroundColor Green
    } catch {
        Write-Host "轉換失敗: $($_.Name) - $($_.Exception.Message)" -ForegroundColor Red
    }
}
```

### 使用 Cursor/VSCode

1. 開啟檔案
2. 查看右下角編碼顯示
3. 如果不是 UTF-8，點擊編碼
4. 選擇「透過編碼儲存」→ `UTF-8`
5. 儲存檔案

## 📋 最佳實踐

1. **始終使用 UTF-8**
   - 所有文字檔案（.md, .txt, .json, .ts, .js 等）都應使用 UTF-8
   - 避免使用 Big5、GB2312 等區域編碼

2. **檢查新建立的檔案**
   - 建立新檔案後，確認右下角顯示 UTF-8
   - 如果顯示其他編碼，立即轉換

3. **版本控制**
   - Git 預設使用 UTF-8，確保檔案編碼正確
   - 避免提交非 UTF-8 編碼的檔案

4. **團隊協作**
   - 確保所有團隊成員都設定 UTF-8
   - 分享 `.editorconfig` 和 `.vscode/settings.json`

## ❓ 常見問題

### Q: 為什麼 Chrome 開啟 Markdown 會亂碼？

A: Chrome 無法自動識別 Markdown 檔案的編碼。建議使用 Cursor/VSCode 的預覽功能，或安裝 Markdown 檢視器擴充功能。

### Q: 如何確認檔案是否為 UTF-8？

A: 在 Cursor/VSCode 中查看右下角的編碼顯示，或使用 PowerShell 腳本檢查。

### Q: 轉換編碼後檔案內容會改變嗎？

A: 不會。UTF-8 可以正確表示所有 Unicode 字元，包括繁體中文。轉換只是改變檔案的編碼格式，內容保持不變。

### Q: Git 提交時出現編碼警告？

A: 確保所有檔案都是 UTF-8 編碼。可以設定 Git 的 `core.autocrlf` 和 `core.quotepath`：

```powershell
git config --global core.autocrlf false
git config --global core.quotepath false
```
